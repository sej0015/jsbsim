      54             : /** Encapsulates an Actuator component for the flight control system.
      55             :     The actuator can be modeled as a "perfect actuator", with the Output
      56             :     being set directly to the input. The actuator can be made more "real"
      57             :     by specifying any/all of the following additional effects that can be
      58             :     applied to the actuator. In order of application to the input signal,
      59             :     these are:
      60             :     
      61             :     - System lag (input lag, really)
      62             :     - Rate limiting
      63             :     - Deadband
      64             :     - Hysteresis (mechanical hysteresis)
      65             :     - Bias (mechanical bias)
      66             :     - Position limiting ("hard stops")
      67             :     
      68             :     There are also several malfunctions that can be applied to the actuator
      69             :     by setting a property to true or false (or 1 or 0).
      70             : 
      71             :     Rate limits can be specified either as a single number or property. If a
      72             :     single <rate_limit> is supplied (with no "sense" attribute) then the
      73             :     actuator is rate limited at +/- the specified rate limit. If the
      74             :     <rate_limit> element is supplied with a "sense" attribute of either
      75             :     "incr[easing]" or "decr[easing]" then the actuator is limited to the
      76             :     provided numeric or property value) exactly as provided.
      77             :     
      78             :     Lag filter's numerical integration assumes that the lag parameter is
      79             :     constant. So a continuously varying parameter via a property will introduce
      80             :     a non negligible error that will accumulate as the simulation progresses.
      81             : 
      82             : Syntax:
      83             : 
      84             : @code
      85             : <actuator name="name">
      86             :   <input> {[-]property} </input>
      87             :   <lag> {property name | value} </lag>
      88             :   [<rate_limit> {property name | value} </rate_limit>]
      89             :   [<rate_limit sense="incr"> {property name | value} </rate_limit>
      90             :    <rate_limit sense="decr"> {property name | value} </rate_limit>]
      91             :   <bias> number </bias>
      92             :   <deadband_width> number </deadband_width>
      93             :   <hysteresis_width> number </hysteresis_width>
      94             :   [<clipto>
      95             :     <min> {property name | value} </min>
      96             :     <max> {property name | value} </max>
      97             :    </clipto>]
      98             :   [<output> {property} </output>]
      99             : </actuator>
     100             : @endcode
     101             :
