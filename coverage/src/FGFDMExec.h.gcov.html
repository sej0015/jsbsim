      79             : /** Encapsulates the JSBSim simulation executive.
      80             :     This class is the executive class through which all other simulation classes
      81             :     are instantiated, initialized, and run. When integrated with FlightGear (or
      82             :     other flight simulator) this class is typically instantiated by an interface
      83             :     class on the simulator side.
      84             : 
      85             :     At the time of simulation initialization, the interface
      86             :     class creates an instance of this executive class. The
      87             :     executive is subsequently directed to load the chosen aircraft specification
      88             :     file:
      89             : 
      90             :     @code{.cpp}
      91             :     fdmex = new FGFDMExec( ... );
      92             :     result = fdmex->LoadModel( ... );
      93             :     @endcode
      94             : 
      95             :     When an aircraft model is loaded, the config file is parsed and for each of
      96             :     the sections of the config file (propulsion, flight control, etc.) the
      97             :     corresponding Load() method is called (e.g. FGFCS::Load()).
      98             : 
      99             :     Subsequent to the creation of the executive and loading of the model,
     100             :     initialization is performed. Initialization involves copying control inputs
     101             :     into the appropriate JSBSim data storage locations, configuring it for the
     102             :     set of user supplied initial conditions, and then copying state variables
     103             :     from JSBSim. The state variables are used to drive the instrument displays
     104             :     and to place the vehicle model in world space for visual rendering:
     105             : 
     106             :     @code{.cpp}
     107             :     copy_to_JSBsim(); // copy control inputs to JSBSim
     108             :     fdmex->RunIC(); // loop JSBSim once w/o integrating
     109             :     copy_from_JSBsim(); // update the bus
     110             :     @endcode
     111             : 
     112             :     Once initialization is complete, cyclic execution proceeds:
     113             : 
     114             :     @code{.cpp}
     115             :     copy_to_JSBsim(); // copy control inputs to JSBSim
     116             :     fdmex->Run(); // execute JSBSim
     117             :     copy_from_JSBsim(); // update the bus
     118             :     @endcode
     119             : 
     120             :     JSBSim can be used in a standalone mode by creating a compact stub program
     121             :     that effectively performs the same progression of steps as outlined above
     122             :     for the integrated version, but with two exceptions. First, the
     123             :     copy_to_JSBSim() and copy_from_JSBSim() functions are not used because the
     124             :     control inputs are handled directly by the scripting facilities and outputs
     125             :     are handled by the output (data logging) class. Second, the name of a script
     126             :     file can be supplied to the stub program. Scripting (see FGScript) provides
     127             :     a way to supply command inputs to the simulation:
     128             : 
     129             :     @code{.cpp}
     130             :     FDMExec = new JSBSim::FGFDMExec();
     131             :     FDMExec->LoadScript( ScriptName ); // the script loads the aircraft and ICs
     132             :     result = FDMExec->Run();
     133             :     while (result) { // cyclic execution
     134             :       result = FDMExec->Run(); // execute JSBSim
     135             :     }
     136             :     @endcode
     137             : 
     138             :     The standalone mode has been useful for verifying changes before committing
     139             :     updates to the source code repository. It is also useful for running sets of
     140             :     tests that reveal some aspects of simulated aircraft performance, such as
     141             :     range, time-to-climb, takeoff distance, etc.
     142             : 
     143             :     <h3>JSBSim Debugging Directives</h3>
     144             : 
     145             :     This describes to any interested entity the debug level
     146             :     requested by setting the JSBSIM_DEBUG environment variable.
     147             :     The bitmasked value choices are as follows:
     148             :     - <b>unset</b>: In this case (the default) JSBSim would only print
     149             :        out the normally expected messages, essentially echoing
     150             :        the config files as they are read. If the environment
     151             :        variable is not set, debug_lvl is set to 1 internally
     152             :     - <b>0</b>: This requests JSBSim not to output any messages
     153             :        whatsoever
     154             :     - <b>1</b>: This value explicity requests the normal JSBSim
     155             :        startup messages
     156             :     - <b>2</b>: This value asks for a message to be printed out when
     157             :        a class is instantiated
     158             :     - <b>4</b>: When this value is set, a message is displayed when a
     159             :        FGModel object executes its Run() method
     160             :     - <b>8</b>: When this value is set, various runtime state variables
     161             :        are printed out periodically
     162             :     - <b>16</b>: When set various parameters are sanity checked and
     163             :        a message is printed out when they go out of bounds
     164             : 
     165             :     <h3>Properties</h3>
     166             :     @property simulator/do_trim (write only) Can be set to the integer equivalent to one of
     167             :                                 tLongitudinal (0), tFull (1), tGround (2), tPullup (3),
     168             :                                 tCustom (4), tTurn (5). Setting this to a legal value
     169             :                                 (such as by a script) causes a trim to be performed. This
     170             :                                 property actually maps toa function call of DoTrim().
     171             : 
     172             :     @author Jon S. Berndt
     173             :     @version $Revision: 1.106 $
     174             : */
